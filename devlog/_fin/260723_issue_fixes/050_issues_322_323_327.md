# Issues #322, #323, and #327

## #322 Codex shim subcommand detection

[Decision Log]
- 목적과 의도: Codex global flag 뒤의 `app-server` 같은 management subcommand가 proxy autostart를 다시 켜지 않도록 한다.
- 기존 구현 및 제약 조건: sh는 `$1`, cmd는 `%~1`, PowerShell은 `$firstArg`만 검사했다. 옵션 값은 positional token처럼 보일 수 있고 세 wrapper의 동작이 같아야 한다.
- 검토한 주요 대안: 모든 인자에서 internal command 검색, 첫 non-flag 검색, 값 소비 옵션을 건너뛴 첫 positional token 검색.
- 선택한 방식: 현재 Codex global value option 목록을 건너뛰고 첫 positional token만 subcommand로 판별한다.
- 다른 대안 대신 이 방식을 선택한 이유: 모든 인자 검색은 prompt/하위 command 인자를 오인하고, 첫 non-flag 방식은 `read-only` 같은 옵션 값을 subcommand로 오인한다.
- 장점, 단점 및 영향: flags-before-command를 세 OS wrapper에서 동일하게 처리한다. Codex에 새 value option이 추가되면 목록 동기화가 필요하다.

## #323 reasoning summary compatibility

[Decision Log]
- 목적과 의도: 일부 Responses-compatible 모델이 거부하는 `reasoning_summary_delivery`를 모델별로 비활성화한다.
- 기존 구현 및 제약 조건: unknown routed model은 summary 지원을 기본 `true`로 광고하고 adapter는 request body를 그대로 전달한다. 전역 default 변경은 정상 provider 기능을 불필요하게 낮춘다.
- 검토한 주요 대안: 전역 default false, enum 전역 변환, 모델별 capability override와 adapter guard.
- 선택한 방식: `modelSupportsReasoningSummaries` map을 catalog metadata에 반영하고 명시적 false 모델의 summary 필드를 adapter에서도 제거한다.
- 다른 대안 대신 이 방식을 선택한 이유: upstream별 enum 지원이 다르므로 전역 변환은 안전하지 않고, adapter guard가 stale catalog client도 보호한다.
- 장점, 단점 및 영향: 기존 설정은 byte-for-byte behavior를 유지한다. 호환성 문제가 있는 모델만 config opt-out이 필요하다.

## #327 main Codex account auth health

[Decision Log]
- 목적과 의도: native `__main__` 계정의 missing/terminal-invalid credential 상태를 dashboard와 pool rotation에 동일하게 노출한다.
- 기존 구현 및 제약 조건: main WHAM non-2xx는 전부 null quota로 축약됐고 DTO의 `hasCredential`은 항상 true였다. 일반 403을 terminal auth failure로 오판하면 정상 계정을 불필요하게 제외할 수 있다.
- 검토한 주요 대안: 모든 401/403 mark, DTO-only 표시, 401 및 allowlisted structured 403 code만 runtime mark.
- 선택한 방식: missing credential와 401, `invalid_workspace_selected`/`invalid_refresh_token`만 `needsReauth`로 mark하고 성공 refresh에서 clear한다.
- 다른 대안 대신 이 방식을 선택한 이유: 실제 routing eligibility와 UI 상태를 맞추면서 generic permission failure의 false positive를 피한다.
- 장점, 단점 및 영향: main과 pool의 관측성이 일치하고 invalid main이 rotation에서 제외된다. 새 terminal 403 code는 명시적으로 allowlist해야 한다.
