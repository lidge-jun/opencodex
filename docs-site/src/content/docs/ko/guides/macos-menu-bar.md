macOS 12 이상에서 메뉴바 앱을 열면 시스템 상단 메뉴바에서 프록시 상태를 바로 확인할 수 있습니다.
버전, 가동 시간, PID, 포트, Bun 런타임 출처, launchd 서비스 상태를 보여 주고 터미널 없이 주요 동작을
실행합니다.

## 빌드하고 열기

먼저 `ocx` CLI를 설치하고 초기화한 뒤 저장소에서 앱을 빌드합니다.

```bash
bun run build:macos-app
open "dist/macos/OpenCodex.app"
```

기본 빌드는 현재 Mac 아키텍처를 대상으로 합니다. 전체 Xcode toolchain이 선택된 환경에서 arm64와
x86_64를 함께 넣으려면 다음처럼 빌드합니다. Command Line Tools만으로는 현재 아키텍처만 빌드할 수 있습니다.

```bash
UNIVERSAL=1 bun run build:macos-app
```

로컬 실행용 ad-hoc 서명까지 자동으로 적용됩니다. 경고 없이 외부에 배포하려면 메인테이너의
Developer ID 서명과 notarization 절차가 별도로 필요합니다.

## 릴리즈 패키징

Release 워크플로는 arm64와 x86_64를 함께 빌드하고 `OpenCodex.app`을
`OpenCodex-<version>-macos-universal.zip`으로 압축한 뒤 대응하는 `.sha256` 파일을 생성합니다.
드라이런에서도 두 파일을 워크플로 아티팩트로 전달하고 검증합니다. 실제 릴리즈에서는 npm 배포가
성공한 후 두 파일을 GitHub Release에 첨부합니다.

자동 생성된 압축 파일은 ad-hoc 서명 상태이며 notarization되지 않으므로, 첫 실행 때 macOS에서
수동 승인이 필요할 수 있습니다.

## 메뉴 동작

| 항목 | 동작 |
| --- | --- |
| **프록시 상태** | 항상 선명하게 표시되며, 서브메뉴에서 버전, PID, 포트, 가동 시간, 런타임, 서비스, CLI 상세 정보를 확인합니다. |
| **대시보드 열기** | `ocx status --json`이 알려 준 현재 대시보드 URL을 엽니다. |
| **프록시 시작** | launchd 서비스가 설치되어 있으면 서비스를 시작하고, 아니면 단독 프록시를 실행합니다. |
| **프록시 재시작** | 확인 후 서비스 설치 상태에 맞는 종료/시작 순서를 실행합니다. |
| **프록시 종료** | 확인 후 요청을 정상적으로 정리하고 프록시와 서비스를 멈춘 뒤 네이티브 Codex 설정을 복원합니다. |
| **상태 새로고침** | CLI 상태를 즉시 다시 읽습니다. 메뉴를 열 때와 주기적 타이머에서도 자동 갱신됩니다. |
| **메뉴바 앱 종료** | 메뉴바 앱만 닫습니다. 프록시 상태는 바뀌지 않습니다. |

재시작과 종료는 진행 중 요청을 끊을 수 있으므로 확인창을 거칩니다.

## `ocx` 찾기

앱은 상속된 `PATH`, Homebrew, Bun, Volta, pnpm, `~/.local/bin`, nvm, fnm 위치를 자동 탐색합니다.
버전 관리자가 GUI 앱에서 실행 파일을 숨기는 경우 **ocx CLI 선택…**에서 파일을 직접 고를 수 있습니다.
**CLI 자동 탐색 사용**을 누르면 이 경로 지정이 해제됩니다.

launchd 서비스가 사용자 정의 `CODEX_HOME` 또는 `OPENCODEX_HOME`으로 설치되었다면 앱도 해당 plist의
두 경로를 이어받습니다. API 키, OAuth 인증 정보, 요청 본문, 계정 식별자는 읽거나 표시하지 않습니다.

앱 자체도 로그인할 때 열려야 한다면 **시스템 설정 → 일반 → 로그인 항목**에 빌드된 앱을 추가하세요.
이 설정은 `ocx service install`과 역할이 다릅니다. 로그인 항목은 메뉴 UI를 열고, launchd 서비스는
프록시 상시 실행과 비정상 종료 후 재시작을 담당합니다.
